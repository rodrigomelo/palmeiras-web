# Palmeiras Agenda iOS

Native iOS client scaffold for the shared Palmeiras Agenda backend.

The app must call the same backend as Web and Android:

```text
https://palmeiras.rodrigolanna.com.br/api/v1
```

The client also retries the legacy `/api` route for compatibility with older
links and clients.

Current contents:

- `AppRootView.swift` - root view to use from your generated SwiftUI app entry point
- `HomeView.swift` - minimal native agenda screen
- `PalmeirasAPIClient.swift` - backend client
- `Models.swift` - DTOs matching `packages/contracts/openapi.yaml`
- `AppConfiguration.swift` - API base URL configuration
- `Assets.xcassets/AppIcon.appiconset` - generated PA calendar app icons

Current shared app/API version: `1.1.37`. Keep the Xcode target marketing
version and `AppConfiguration.appVersion` aligned with `APP_VERSION` and
`packages/contracts/openapi.yaml`.

To turn this into an Xcode project, create a new SwiftUI iOS app named
`PalmeirasAgenda`, add these Swift files and `Assets.xcassets` to the app target, and set the bundle
identifier/versioning in Xcode. In the app file generated by Xcode, use:

```swift
import SwiftUI

@main
struct PalmeirasAgendaApp: App {
    var body: some Scene {
        WindowGroup {
            AppRootView()
        }
    }
}
```

Do not add a second `@main` app file. Keep all network models generated from
`packages/contracts/openapi.yaml` as the API expands.

Refresh the app icon catalog from the shared brand source with:

```bash
scripts/export-brand-assets.py
```

## React Native error: "No script URL provided"

This repository's iOS scaffold is native SwiftUI and does not load JavaScript.
If Xcode shows `No script URL provided`, you are launching a different React
Native iOS target. Fix that target by making sure its `AppDelegate` returns a
bundle URL in both Debug and Release.

For a modern React Native `AppDelegate.mm`:

```objc
- (NSURL *)sourceURLForBridge:(RCTBridge *)bridge
{
#if DEBUG
  return [[RCTBundleURLProvider sharedSettings] jsBundleURLForBundleRoot:@"index"];
#else
  return [[NSBundle mainBundle] URLForResource:@"main" withExtension:@"jsbundle"];
#endif
}
```

For older templates that create `RCTRootView` directly:

```objc
NSURL *jsCodeLocation;
#if DEBUG
jsCodeLocation = [[RCTBundleURLProvider sharedSettings] jsBundleURLForBundleRoot:@"index"];
#else
jsCodeLocation = [[NSBundle mainBundle] URLForResource:@"main" withExtension:@"jsbundle"];
#endif
```

For Release builds, also confirm the Xcode target has the React Native bundle
phase, usually named `Bundle React Native code and images`, so
`main.jsbundle` is copied into the app.
