🔑 Key Points

✅ Single database, two servers. Both production (Vercel) and local (server.py) read from the same Supabase instance.

✅ Collectors write to Supabase. Cron runs on the Mac mini, fetches data, pushes to Supabase. Both servers serve fresh data on next request.

✅ No Python dependencies on Vercel. All api/*.py files use urllib (stdlib) for Supabase REST calls — no supabase library needed.

✅ Same domain, no CORS. Frontend and API are on the same URL — /api/* routes are relative.

✅ Local = server.py on port 5001. Managed by launchd, auto-starts on boot. Serves static files + API routes.
