🔄 How Updates Flow
Yes — Supabase updates are reflected on BOTH local and production.
Both local (server.py) and Vercel (api/*.py) read from the same Supabase database. When collectors update Supabase, both servers serve the new data on the next API call.
🕐 Cron triggers collector
→
Collector fetches from APIs
→
Writes to Supabase
→
Local server.py reads Supabase
+
Vercel api/*.py reads Supabase
→
Frontend displays fresh data

🚀 Deployment
Environment URL How it runs API backend
Production palmeiras-web.vercel.app Vercel serverless api/*.py → Supabase REST (urllib)
Local Dev 127.0.0.1:5001 server.py (launchd, auto-start) server.py → Supabase REST (urllib)
Both environments read from the SAME Supabase database.
Updates to Supabase (via collectors) are immediately available on the next API call from either environment.

Local dev uses server.py on port 5001 — runs automatically via macOS launchd. Serves static files + API routes. No Vercel CLI needed.
vercel dev is only for debugging Vercel-specific issues before deploying — not for daily use.
