⚽ Palmeiras Web — System Diagram

Architecture Overview • March 2026

🔑 Key Points

✅ Single database, two servers. Both production (Vercel) and local (server.py) read from the same Supabase instance.

✅ Collectors write to Supabase every 4h. Cron runs via OpenClaw, fetches from APIs, pushes to Supabase. Both servers serve fresh data immediately.

✅ No Python dependencies on Vercel. All api/*.py use urllib (stdlib) for Supabase REST — no supabase library.

✅ Same domain, no CORS. Frontend calls /api/* on same domain.

✅ Local dev = server.py on :5001 (launchd). Static files + API routes. No Vercel CLI needed.

🛠️ Tech Stack

Frontend: Vanilla HTML/CSS/JS + Chart.js v4

API: Python 3 (BaseHTTPRequestHandler, stdlib only)

Database: Supabase (PostgreSQL) via REST API

Hosting: Vercel serverless (production) + server.py (local)

Collectors: Python scripts + OpenClaw cron (every 4h)
